# Aifeels Trademark Policy

**Version:** 1.0  
**Last Updated:** 2025-02-07

---

## Overview

The "Aifeels" name and logo are trademarks used to identify the official Aifeels specification and certified implementations.

This policy ensures that the Aifeels mark remains associated with quality, conformance, and vendor neutrality.

---

## Trademark Ownership

**Trademark:** "Aifeels"  
**Owner:** Aifeels Project (maintained by specification maintainers)  
**Status:** Unregistered (common law trademark)

The Aifeels Project does not currently seek formal trademark registration. This policy governs usage under common law trademark principles.

---

## Permitted Uses (No Permission Required)

You may use "Aifeels" without permission for:

### ✅ Reference and Discussion
- Accurate statements about the specification
- Documentation referring to Aifeels
- Academic papers and research
- Blog posts and tutorials
- Conference talks and presentations

**Example:** "Our agent implements the Aifeels emotional state specification."

### ✅ Compatibility Claims
- Stating that your implementation "implements Aifeels v0.1"
- Claiming "Aifeels-compatible" or "Aifeels-compliant"
- Using "Aifeels" in comparison charts

**Requirement:** Must be accurate and verifiable.

### ✅ Implementation Names
- Using "Aifeels" as part of your implementation name
- Examples: "aifeels-python", "aifeels-js", "rust-aifeels"

**Requirements:**
- Implementation must actually implement the spec
- Name must not imply official endorsement
- Cannot use just "Aifeels" alone (must have qualifier)

### ✅ Community Resources
- User groups: "Aifeels Users Group"
- Forums: "Aifeels Discussion Forum"
- Unofficial docs: "Unofficial Aifeels Guide"

**Requirement:** Must clearly state "unofficial" or "community" if not endorsed by maintainers.

---

## Uses Requiring Permission

You must request permission (trademark@aifeels.org) for:

### ⚠️ Commercial Products
- Products with "Aifeels" in the product name
- Commercial services branded as "Aifeels [Service]"
- Hosted platforms using "Aifeels" in domain name

**Example:** "Aifeels Cloud Platform" would require permission.

**Likely granted if:**
- Implements specification correctly
- Does not imply official endorsement
- Maintains vendor neutrality

### ⚠️ Certification Marks
- Using "Aifeels Certified" badge without passing certification
- Creating derivative certification programs
- Implying official endorsement without certification

**See:** [CERTIFICATION.md](CERTIFICATION.md) for proper certification process.

### ⚠️ Events and Conferences
- Events named "Aifeels Conference" or similar
- Official-sounding event names using "Aifeels"

**Likely granted if:**
- Event is open and non-exclusive
- No false endorsement implied
- Community-focused

### ⚠️ Modifications
- Modified specifications branded as "Aifeels"
- Forks of the specification using "Aifeels"
- Extensions claiming to be "official Aifeels"

**Generally not granted.** Use different name for forks/modifications.

---

## Prohibited Uses

You may NOT use "Aifeels" for:

### ❌ False Association
- Implying official endorsement when none exists
- Suggesting maintainer approval without it
- Creating confusion about project affiliation

### ❌ Quality Misrepresentation
- Non-conformant implementations claiming conformance
- Failed implementations claiming certification
- Misleading compatibility claims

### ❌ Vendor Lock-In
- Using trademark to create proprietary extensions
- Claiming "Aifeels" for vendor-specific features
- Preventing interoperability

### ❌ Disparagement
- Using "Aifeels" to disparage the project
- Associating "Aifeels" with harmful content
- Trademark dilution or tarnishment

## Certification Badge

The **"Aifeels v0.1 Certified"** badge is a certification mark.

### Usage Rules

**Permitted:**
- Only by implementations that have passed certification
- Must link to certification record
- Must match certified version

**Not Permitted:**
- Self-certification without maintainer approval
- Using badge for non-certified implementations
- Modifying badge design

**See:** [CERTIFICATION.md](CERTIFICATION.md) for certification process.

## Enforcement

### Compliance Monitoring

Maintainers periodically review trademark usage for:
- False certification claims
- Vendor neutrality violations
- Quality misrepresentation

### Violation Response

**Step 1: Notification**
- Informal contact requesting correction
- 30-day remedy period

**Step 2: Formal Notice**
- Written cease-and-desist if no response
- 14-day final remedy period

**Step 3: Legal Action**
- Last resort for serious violations
- Rarely necessary

### Good Faith Policy

We assume good faith. Most violations are accidental and easily corrected.

**Example:** Using "Aifeels Certified" badge before certification is complete → Just remove badge and complete certification process.
